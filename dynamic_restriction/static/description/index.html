Dynamic Restrictions

Block risky actions where standard access rights are too broad.

Standard Odoo security is powerful, but business teams often need action-level restrictions that change by state, user, company, or context. Dynamic Access Manager adds that control without model-specific custom code.

  • Prevent create, edit, delete, duplicate, archive, import, and export actions.
  • Protect critical workflows after records reach confirmed, posted, or closed states.
  • Keep backend validation active even when users bypass the web interface.

Group Restrictions

Apply rules to entire teams. For example, allow sales managers to edit confirmed sales orders while preventing salespersons from changing them after confirmation.

Company Restrictions

Secure multi-company environments by applying restrictions only where they are relevant. Company-specific rules keep local policies independent.

Domain / State Restrictions

Use domain conditions to restrict only matching records, such as posted invoices, won opportunities, validated stock pickings, or confirmed purchase orders.

Own Documents Only

Restrict users to documents they own by selecting a responsible user field. This is useful for sales teams, support agents, recruiters, and account managers.

Approval Workflow

Convert high-risk actions into approval requests. Approvers can review the request, execute the approved action, or reject it according to company policy.

Audit Logs

Every blocked operation can be reviewed later with clear context: user, action, model, record, company, restriction, and reason.

UI Button Hiding

Improve usability by hiding restricted actions from the Odoo backend. Users see only the actions they are expected to perform.

Readonly Fields

Protect sensitive fields such as price, discount, salesperson, payment terms, or analytic values while allowing safe fields to remain editable.

Advanced Protection

Control when, where, and how sensitive actions are allowed.

Time Restrictions

Limit risky actions to approved workdays and time windows. Useful for closing periods, after-hours controls, and operational governance.

IP Restrictions

Allow sensitive operations only from trusted office IP addresses or approved network ranges.

Import / Export Protection

Prevent unauthorized imports and exports to protect master data, customer lists, financial information, and confidential business records.

Mass Action Protection

Prevent bulk edit, delete, and archive operations that could impact large record sets from list views.

Restriction Templates

Package repeatable controls into templates and apply them quickly across business units or implementation projects.

Backend First

UI hiding improves experience, but backend validation remains the enforcement layer for RPC, import, automation, and direct calls.
